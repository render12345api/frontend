# SMS Burst Unified Deployment Guide

This project combines the Next.js frontend and the Flask API server into a single deployable unit.

## Prerequisites
- A PostgreSQL database (e.g., Supabase, Neon, or Render PostgreSQL)
- A Render account

## Deployment Steps

### 1. Build the Frontend
The frontend needs to be exported as static files so Flask can serve them.
```bash
pnpm install
pnpm build
```
This will create an `out` directory.

### 2. Deploy to Render
1. Create a new **Web Service** on Render.
2. Connect your repository.
3. Set the following configurations:
   - **Runtime**: `Python`
   - **Build Command**: `pip install -r requirements.txt` (Note: You should have the `out` directory in your repo, or build it as part of the build command if you have Node.js available)
   - **Start Command**: `gunicorn app:app`
4. Add Environment Variables:
   - `DATABASE_URL`: Your PostgreSQL connection string
   - `MASTER_API_KEY`: Your desired admin key (e.g., `smsburst12345`)
   - `PORT`: `10000` (default)

## Project Structure
- `app.py`: Unified Flask server (serves API and Frontend)
- `out/`: Static frontend files (generated by `next build`)
- `app/`: Next.js source code
- `lib/`: Shared utilities and database logic
- `requirements.txt`: Python dependencies
